## 2026-02-04 UI設計変更
### 例外設定時の活動係数非表示制御を取りやめ
- 前回のUI設計変更で、例外設定時に活動係数を非表示にする仕様としたが、
  例外設定時もユーザーが算出方法を切り替え可能なため常時表示に変更
- `EnergyOrderType` が`体重補正代謝量×係数`の時だけ活動係数を表示する方法も

## 2026-02-03 UI設計変更
### 疾患セレクトの位置づけ整理
- 疾患セレクトは「疾患入力」ではなく、計算ロジック上の例外指定である
- 誤解を避けるため、表記を「疾患」→「例外設定」に変更

### 補足表示の簡略化
- 一般的注意書きおよび小児向け項目は、使用者向けUIとして冗長と判断し削除
- 例外設定が選択された場合のみ、疾患別の目安を常時表示する仕様に変更
- 折りたたみは維持せず、判断に必要な情報を即時視認可能とした

### 不要係数の非表示制御
- 例外設定時には活動係数が計算に関与しないため、
  `data-factor-block` でラップし JS で非表示制御を行う
- ストレス係数・褥瘡は蛋白計算に関与するため表示を維持

### 蛋白補正の手動選択状態の可視化
- 手動選択フラグは内部状態として保持
- UIの複雑化を避けるため pill 表示は行わず、
  PCでのみ気付ける tooltip により状態を補足表示

## 2026-01-21 UI設計変更
### 採用体重表示箇所の変更
- 基礎代謝量計算は実測体重、エネルギー計算では採用体重（実測・標準・調整）を使用しているため、
  採用体重は右列基礎代謝量欄から、左列必要エネルギー欄に移動
- 左列算出法の「基礎代謝量×係数」を「体重補正代謝量×係数」に変更：
  理論上の基礎代謝量とエネルギー計算に使用する代謝量をUIでも区別

### フォーム再送信確認の回避
- スマホで戻るボタン押下時に表示される「フォーム再送信確認」ダイアログを回避するため、
  フォーム送信をAJAX化
- スマホで戻るボタンは終了の意図と思われるのでそのまま閉じる仕様

### IEで左列・右列幅を調整
- IEで左列幅を52.5%→48%に変更し、左右バランスをモダンブラウザと同等に
